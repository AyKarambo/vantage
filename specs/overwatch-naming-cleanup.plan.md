# Techplan: `overwatch-naming-cleanup`

Implements [`overwatch-naming-cleanup.spec.md`](overwatch-naming-cleanup.spec.md). Rename every
"Overwatch 2" / "OW2" reference to "Overwatch" / "OW" across docs, UI copy, code comments, and the
spec archive — **text-only, behaviour-preserving**, keeping the numeric Overwolf game id and every
protocol string untouched.

## Architecture & Approach

This is a **mechanical text rename**, but a blind repo-wide find/replace is unsafe — several
substring matches are load-bearing and must survive (lockfile integrity hash, test-fixture
variable names, an external URL slug, the game id, GEP protocol keys, and an HTML `&nbsp;`
entity). The approach is therefore a **targeted, inventory-driven edit**, not a global sweep:

1. **Worklist = the verified inventory** (57 line-edits across 32 files, enumerated below). Each
   edit was line-located and classified by the Explore survey; the implementer applies exactly
   those, using the uniform replacement rule plus the four special-case handlers.
2. **Uniform replacement rule**
   - Full phrase `Overwatch 2` → `Overwatch` (drop the ` 2`; in HTML drop `&nbsp;2`).
   - Standalone `OW2` → `Overwatch` in reader-facing prose (README, docs, specs); → `OW` in
     dense code comments where the abbreviation register already lived (rank engine/types,
     season). Never leave an `OW2`. The exact per-line choice is pinned in the tables below so
     `/implement` is pure execution.
3. **Special-case handlers** (must not be done by naive replace):
   - **ASCII diagram re-pad** — `README.md:306`.
   - **HTML `&nbsp;2`** — legal pages: remove the entity+digit, not just the `2`.
   - **URL preservation** — `editable-master-data.spec.md:117`: change link *text* only, keep the
     `overwatch-2-api` URL target.
   - **Double-token collapse** — `supercompensation-detection.spec.md:166`: `OW2/OW` → single
     `Overwatch`.
4. **Protected set — never touched** (see the Do-Not-Touch table). A `git diff --stat` gate
   confirms `package-lock.json` and `test/notionRuntime.test.ts` are absent from the diff.
5. **Verify** — regression grep (zero remaining variants outside exclusions) + `npm run
   typecheck` + `npm test`, then rebuild the renderer bundle and eyeball the two changed UI
   strings and the two ASCII diagrams / three legal pages.

No new modules, no scripted migration required. Optionally, the ~40 *uniform* prose/comment lines
could be done with a pattern-scoped script (literal `Overwatch 2`, excluding `*.map`,
`package-lock.json`, and the URL line), but given the special cases the primary path is direct
edits.

  About tagline and onboarding lead render "Overwatch" with no "2" (preview harness at

- **AC3 / AC4 — id & protocol preserved.** Diff review confirms `10844`, GEP keys, and the SDK

## Risks & Alternatives

| Risk | Mitigation |
|---|---|
| Blind global replace corrupts lockfile hash / test vars / URL / game id / GEP keys / `&nbsp;` | Inventory-driven targeted edits + explicit Do-Not-Touch table + post-edit grep and `git diff --stat` gate |
| ASCII diagram misalignment (`README:306`) | Re-pad `┐` by +2 dashes; verify in monospace (AC5) |
| HTML: replacing only `2` leaves a stray `&nbsp;` | Handler removes the whole `&nbsp;2` sequence |
| `OW2/OW` double token reads as `Overwatch/OW` | Explicit collapse to a single `Overwatch` (supercompensation:166) |
| Generated preview bundle still shows old copy pre-rebuild | It's gitignored and regenerates on build — noted, no action; renderer source is the source of truth |
| Dropping the ``overwatch 2`` store keyword slightly hurts search discoverability | Accepted per spec resolved-question (consistency > legacy keyword); reversible later if store analytics warrant |
| Inconsistent `OW2`→`OW` vs `→Overwatch` choices | Per-line choice pinned in the tables above; AC1 only requires zero `OW2`, so either is valid |

**Alternatives considered**
- *Scripted pattern-scoped sed for the uniform lines.* Viable accelerator for the ~40 plain
  `Overwatch 2` prose/comment lines (exclude `*.map`, `package-lock.json`, URL line), with the 4
  special cases done by hand. Kept as an option, not the primary path, to avoid scripting risk on
  a one-shot change.
- *Leave the spec archive untouched (living surfaces only).* Rejected — the spec's resolved
  questions chose "everything, including specs".
- *Keep "Overwatch 2" where it named the Overwolf-registered target / store listing.* Rejected —
  resolved question chose "normalize everywhere", id retained.
